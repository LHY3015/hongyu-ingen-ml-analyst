## Overview

MDP schema for the Aido Rover patrol / anomaly-response task. `rover_transitions.csv` is generated by rolling a scripted behaviour policy through the world core (blockages ON) — the same core the W05 Gymnasium env wraps, so offline data and the online env share identical dynamics. Feeds Phase C RL experiments. Full design rationale and calibration evidence: `reports/W02_RL_scaffold.md`.

## State Space  S

| # | Feature              | Description                                                   |
| - | -------------------- | ------------------------------------------------------------- |
| 0 | torque_mean          | Mean wheel torque over last 50 steps (Nm)                     |
| 1 | torque_max           | Peak torque over last 50 steps (Nm)                           |
| 2 | torque_std           | Torque variability over last 50 steps                         |
| 3 | lidar_mean           | Mean LiDAR distance over last 50 steps (m)                    |
| 4 | soc_slope            | Battery discharge rate over last 50 steps (%/step)            |
| 5 | battery_soc          | Current battery state of charge (%)                           |
| 6 | route_progress       | Fraction of the main patrol loop completed (0-1)              |
| 7 | next_main_block_dist | Distance (m) to next blockage if continuing on the main route |
| 8 | branch_block_dist    | Distance (m) to next blockage if rerouting onto the branch    |

No absolute GPS (position leakage: fixed terrain -> faults at fixed coordinates)

## Action Space  A

| ID | Name           | Effect in RoverWorld                                  |
| -- | -------------- | ----------------------------------------------------- |
| 0  | continue       | normal speed, route unchanged                         |
| 1  | slow           | speed x0.5, scales down the fault-trigger probability |
| 2  | reroute        | take the branch at the next node                      |
| 3  | raise-alert    | no motion effect in the core                          |
| 4  | return-to-base | no motion effect in the core; <br />episode ends     |

## Reward Function  R

| Label   | Condition                               | Action         | Reward                                                 |
| ------- | --------------------------------------- | -------------- | ------------------------------------------------------ |
| normal  | not halted                              | continue       | +1.0                                                   |
| normal  | halted                                  | continue       | -0.5 (closes the "idle at a block still pays" exploit) |
| normal  | low-slip terrain                        | slow           | -0.1                                                   |
| normal  | high-slip terrain (torque_mean > 30 Nm) | slow           | 0.0 (justified proactive slow-down)                    |
| normal  | main route clear                        | reroute        | -0.3                                                   |
| normal  | main route blocked                      | reroute        | 0.0 (justified detour)                                 |
| normal  | --                                      | raise-alert    | -1.5                                                   |
| normal  | --                                      | return-to-base | -3.0                                                   |
| anomaly | --                                      | continue       | -5.0                                                   |
| anomaly | --                                      | slow           | +1.5 (real fault-mitigating effect in the core)        |
| anomaly | --                                      | reroute        | 0.0 (ineffective against a self-fault)                 |
| anomaly | --                                      | raise-alert    | +5.0                                                   |
| anomaly | --                                      | return-to-base | +3.0                                                   |
| any     | --                                      | any            | -0.05 per step (energy)                                |

Shaping: +2.0 when `battery_soc < 20.0%`  AND `action == return-to-base`.

## Discount

gamma = 0.99 (anchored to PIC 2.0 GRPO config range γ=0.975–0.995)

## Scripted Behaviour Policy (baseline; to log transition table)

Priority-ordered rule table (first matching rule wins), + 5% epsilon-explore over {continue, slow, reroute, raise-alert} (exploration never randomly samples return-to-base — an episode only ends on a genuine scripted decision):

| Priority | Condition                                                | Action                                |
| -------- | -------------------------------------------------------- | ------------------------------------- |
| 1        | battery_soc < 20.0%                                      | return-to-base (episode ends)         |
| 2        | stuck_counter >= 80 (full block unresolved)              | return-to-base (episode ends, forced) |
| 3        | halted AND main+branch both blocked ahead                | raise-alert (double blockage)         |
| 4        | anomaly_label == 1 (ground truth)                        | raise-alert                           |
| 5        | main route blocked ahead (< 150 m) AND branch clear      | reroute                               |
| 6        | torque_mean > 30 Nm (wet-grass/mud only, high slip risk) | slow                                  |
| 7        | default                                                  | continue                              |

`raise-alert` on fault uses the ground-truth label (competent-expert behaviour-cloning convention) — a torque-std threshold was tried and rejected: confounded by terrain roughness, ~27% false-alert rate on calibration data.

Reroute/full-block trigger distance (150 m) was calibrated empirically against
actual node-crossing traces ;    

`slow`'s trigger (30 Nm) is set to isolate wet-grass/mud (slip weight 0.70/1.00) from dry-grass/gravel (slip weight 0.30/0.10) using window `torque_mean` as a clean, observable proxy (dry_grass 25.2-27.5 Nm vs. wet_grass 36.9+ Nm, a clean gap)   

## Episode Structure

- `MAP_SEED=6`, `HAZARD=0.05`, `blockages=True` (same map as the W2 detection stream; blockages ON for RL )
- Multiple episodes (different blockage/fault seeds) chained to ~48,000 rows; episode cap = 9,600 steps (one full main-loop circuit); ends early only on low battery or an unresolved full block
- Initial `battery_soc`: episode 0 = 100%; subsequent episodes ~ Uniform(40.0, 100.0) (fleet-dispatch realism; without this the 20% auto-dock branch never fires within one loop)
- `episode_id` column enables per-episode return computation in W05

## Known Limitations

- Blockages are static within an episode and `raise-alert` cannot clear one in this core — a genuine full
  block is a scripted dead end resolved only by the stuck-timeout abort. W05 wrapper suggestion: let
  `raise-alert` clear a blockage after N steps (simulated operator response) so alert has a learnable
  positive effect in full-block episodes.
- `route_progress` is an odometer-based approximation (cumulative distance mod loop length); it drifts
  slightly from true DAG position after a branch detour.
